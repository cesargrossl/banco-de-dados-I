O que é um Banco de Dados?

Banco de dados é uma coleção organizada de dados, armazenados de forma estruturada, para que possam ser consultados, gerenciados e utilizados por sistemas e usuários.

Exemplos do dia a dia:
  • Aplicativos bancários;
  • Redes sociais;
  • Lojas online;
  • Sistemas acadêmicos;
  • Sistemas hospitalares;
  • Aplicativos de transporte.
Observação: um banco de dados não precisa ser, obrigatoriamente, informatizado. Porém, os dados precisam possuir organização e significado.

Dado x Informação

Dado

Dado é um fato isolado, sem contexto suficiente para gerar entendimento completo.

Exemplo de dado: Fulano

Informação

Informação é o dado interpretado dentro de um contexto.

Exemplo de informação: Fulano é aluno da Univille

Importância dos Bancos de Dados

Os bancos de dados são fundamentais para as aplicações modernas, pois permitem:

  • Organizar grandes volumes de dados;
  • Facilitar a busca e recuperação de informações;
  • Garantir maior segurança;
  • Evitar perda de dados por meio de backup e recuperação;
  • Apoiar a tomada de decisão nas organizações.

Sistema Gerenciador de Banco de Dados (SGBD)

O SGBD é o software responsável por criar, organizar, manipular e proteger os bancos de dados. Ele permite que usuários e sistemas cadastrem, alterem, excluam e consultem dados de forma controlada.

Classificação dos Bancos de Dados

Banco de Dados Relacional

Organiza os dados em tabelas, compostas por linhas e colunas. É muito utilizado em sistemas administrativos, comerciais e acadêmicos.

Banco de Dados Não Relacional

Também conhecido como NoSQL, pode armazenar dados em formatos como documentos, chave-valor, grafos ou colunas.

Arquitetura de Banco de Dados

A arquitetura de banco de dados define como os dados serão organizados, representados e armazenados dentro de um sistema.

Camada Conceitual

Representa a visão abstrata dos dados, sem se preocupar com detalhes técnicos. É nessa etapa que são identificadas as principais entidades e relacionamentos.

Camada Lógica

Define como os dados serão estruturados em tabelas, atributos e relacionamentos.

Camada Física

Refere-se à forma como os dados serão armazenados fisicamente, considerando desempenho, índices, segurança e estrutura de armazenamento.

Etapas de um Projeto de Banco de Dados

  1. Levantamento de requisitos: entendimento das necessidades do sistema.
  2. Projeto conceitual: criação do modelo conceitual.
  3. Projeto lógico: transformação do modelo conceitual em estrutura lógica.
  4. Projeto físico: implementação técnica no SGBD.
